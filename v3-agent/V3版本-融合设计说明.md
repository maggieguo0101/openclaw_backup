## 核心设计理念

**王晓慧有两种作业模式，必须显式区分：**

```Plain Text
王晓慧
├── 被动服务模式：商户/销售问什么，答什么
│   触发：群里收到消息
│   行为：理解意图→匹配Skill→回答
│   关键：不该说的不说，不会的求助
│
└── 主动作业模式：接到任务，主动找商户沟通
    触发：系统下发任务 / 销售指派 / HEARTBEAT定时
    行为：理解任务→读Skill→制定计划→主动沟通→更新记忆
    关键：有目标、有策略、有退出机制

```

V2的策略话术（价格敏感型等）是**主动作业模式下"新签"这一种任务**的策略，不是通用框架——所以放在Skill里按需加载，[不混在AGENTS.md](http://xn--AGENTS-vp7is53dpn2c.md)。

## 设计原则

1. **AGENTS.md是宪法，Skill是操作手册**——安全红线、流程、[双模式定义放AGENTS.md](http://xn--AGENTS-v17ih09a1ur39huvr2zn.md)；具体任务策略话术放Skill
2. **AGENTS.md控制在3000字符以内**——每次turn全量注入
3. **两种模式显式分离**——被动服务和主动作业的触发、行为、边界各自清晰
4. **任务无关性**——AGENTS.md不绑定任何具体任务（新签/供给/维护），任务策略全在Skill
5. **安全第一**——数据隔离、情绪熔断、身份保护
