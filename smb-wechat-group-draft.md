# SMB 商家企微群运营方案

## 一、背景与目标

### 背景
- SMB 商家体量大（约 50 万），但缺乏专属销售人工维护
- 商家在平台遇到问题（改价、上下架、活动报名等）无人快速响应，影响留存和活跃
- 部分销售已与商家建立企微好友关系，存在天然触达基础
- 陈亮已以"王晓慧"身份维护 200+ 个商家群，是最快可验证的试点场景

### 目标
通过建立"商家 + 销售 + 王晓慧（OpenClaw）"三方企微群，由 OpenClaw 作为智能服务助手，**替代人工完成 SMB 商家的日常线上维护**，降低人力成本、提升商家响应效率。

---

## 二、群结构设计

- **群成员：** 商家（1个）+ 销售 + 王晓慧（OpenClaw 账号）
- **群规模：** 1群1商家
- **商家总量：** 约 50 万，初期试点覆盖已有存量群 + 新建群

---

## 三、群来源与建群方案

### 3.1 存量群接入（最快落地，优先推进）

**适用场景：** 陈亮以王晓慧身份已在群内、且为群主的存量群

| 项目 | 说明 |
|---|---|
| 群数量 | ~200+ 个 |
| 群结构 | 1群1商家 ✅ |
| 王晓慧状态 | 已在群内，是群主 ✅ |
| 需要做的事 | **直接接入 OpenClaw，配置 Skill，即可开始服务** |

> 这批群无需任何建群操作，是验证整套模式的最佳起点。

---

### 3.2 已加好友商家（系统建群）

**适用场景：** 销售企微已与商家互为好友

**流程：**
1. 系统 Job 批量读取销售-商家好友关系数据
2. 自动新建企微群，拉入：商家 + 销售 + 王晓慧账号
3. 触发 OpenClaw 发送入群开场白

**限制：**
- API 建群上限：**< 60 个/天**
- 手动建群总上限：**≤ 500 个**

---

### 3.3 未加好友商家（销售拉群）

**适用场景：** 销售企微与商家尚未建立好友关系

**流程：**
1. **短期方案（优先执行）：** 销售人工主动添加商家企微好友
2. 销售建立好友关系后，拉群（商家 + 销售 + 系统分配的王晓慧账号）
3. 触发 OpenClaw 发送入群开场白

**中期方案（待技术评估）：** 系统通过销售企微账号批量添加商家好友（技术成本较大，需评估可行性与企微合规风险）

---

### 3.4 三类群来源汇总

| 来源 | 数量 | 现状 | 需要做什么 | 优先级 |
|---|---|---|---|---|
| 陈亮存量群 | ~200+ | 王晓慧已在群且是群主 | 直接接入 OpenClaw | 🔴 P0 最快落地 |
| 已加好友-系统建群 | <60/天 | 销售已加好友 | 系统 Job 批量建群 + 拉王晓慧 | P1 |
| 未加好友-销售拉群 | 大规模扩张 | 销售需先加好友 | 销售手动加好友 → 拉群 | P1 |

## 四、王晓慧账号管理策略

| 维度 | 策略 |
|---|---|
| 账号数量 | 现有 40 个，覆盖试点阶段绰绰有余（企微单账号上限 500 群） |
| 分配原则 | 每个账号控制在 100~200 群以内，避免触发企微高频风控 |
| 账号分配方式 | 系统统一分配，销售拉群时指定具体王晓慧账号，禁止销售自行选择 |
| 规模扩大后 | 视覆盖群数量评估是否需要采买更多账号 |
| 风控注意事项 | 发消息频率需控制，避免账号被封；重要账号做好备份 |

## 五、OpenClaw 运营机制

### 5.1 入群开场白（自动触发）

群建立或王晓慧首次接入后，OpenClaw 自动发送开场白，内容包括：
- 自我介绍：身份说明（美团智能服务助手）
- 能力说明：可帮助处理哪些问题（改价、上下架、活动报名、评价管理等）
- 使用方式：有问题直接 @王晓慧

### 5.2 日常服务触发机制

| 触发方式 | 说明 |
|---|---|
| **@王晓慧** | 商家主动提问，OpenClaw 识别意图并调用对应 Skill 处理 |
| **关键词监听** | OpenClaw 被动监听群消息，检测到相关业务问题（如"怎么改价"、"活动怎么报名"）时主动响应 |

### 5.3 可调用的 Skill 能力

**商家服务类（直接服务商家）**

| 业务场景 | Skill | 具体能力 |
|---|---|---|
| 商品管理 | deal-group-assistant | 帮商家生成团单、修改团购、下架商品 |
| 店铺装修 | vg-decoration | 帮商家优化店铺展示页面 |
| 评价管理 | merchant-review-cli | 查看和处理商家评价 |
| 门店信息 | merchant shopinfo | 获取门店基础信息、运营状态 |
| 营销活动 | 营销优惠提报接口（AI友好） | 帮商家报名平台活动、提报立减优惠 |

**销售辅助类（辅助销售跟进）**

| 业务场景 | Skill | 具体能力 |
|---|---|---|
| 门店搜索 | apollo search/shop | 快速查商家私海信息 |
| 团单管理 | apollo supply | 查商家团单列表与状态 |
| 工单创建 | apollo aitask | 给销售创建跟进任务 |
| 拜访记录 | MCP query_visit_data | 查询销售历史拜访记录 |
| 组织信息 | MCP apollo_sales_org | 查销售归属与团队信息 |

### 5.4 销售 vs OpenClaw 分工

| 事项 | 销售负责 | OpenClaw负责 |
|---|---|---|
| 建立关系 | ✅ 加好友、拉群、破冰 | — |
| 入群介绍 | — | ✅ 自动发开场白 |
| 日常答疑 | 兜底（复杂/情感类问题） | ✅ 第一响应（标准化问题） |
| 活动推送 | — | ✅ 批量执行定时推送 |
| 商品操作 | — | ✅ 调用 Skill 直接执行 |
| 签约跟进 | ✅ 强意向商家深度跟进 | ✅ 标准化流程引导（待补充签约 Skill） |

## 七、风险与约束

| 风险项 | 说明 | 应对策略 |
|---|---|---|
| 企微账号封禁 | 王晓慧机器人账号发消息过频可能触发风控 | 单账号群数量控制在 200 以内，设置发消息频率限制 |
| 建群速度瓶颈 | API 建群 <60/天，手动上限 500 | 主要依赖销售手动拉群，系统建群作为补充 |
| 商家不活跃 | 商家进群后无互动 | 开场白 + 定期活动推送激活 |
| 销售抵触 | 担心被 AI 替代或背锅 | 明确分工：OpenClaw 处理标准问题，复杂问题交还销售 |
| 存量群历史记录 | 陈亮存量群商家已有沟通上下文，接入后需避免服务割裂感 | 接入前做好话术衔接，必要时重新发开场白说明升级 |
