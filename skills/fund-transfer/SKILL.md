# 资金转账差额 Skill

## 核心原则

**不动原则**：上次已有的钱在哪个账户就不动，本次只调整差额部分。
- 无账户间调拨，操作最简单
- 只产生三类操作：补充、新增、退款

## 使用流程

### 1. 准备输入 JSON

```json
{
  "title": "资金去向汇总",
  "note": "⚡ 原则：上次已有的钱在哪个账户就不动，本次只调整差额",
  "prev_title": "26.3.9 族兴新材 · 每户722.43元",
  "new_title": "26.3.11 新恒泰 · 每户800元",
  "accounts": {
    "长月": 800,
    "maggie": 800,
    "ziqi": 800
  },
  "prev_funds": {
    "坤": 435,
    "长月": 301.43
  },
  "prev_allocation": {
    "长月账户": {"坤": 396, "长月": 301.43, "长月朋友": 25},
    "maggie账户": {"maggie": 475.43, "maggie朋友": 40, "tutu": 107, "小拉": 55, "徐雪": 45},
    "ziqi账户": {"坤": 39, "tutu": 103, "ziqi": 240.43, "ziqi同事": 125, "coco": 215}
  },
  "new_funds": {
    "坤": 455,
    "长月": 321
  },
  "ownership": {
    "maggie": "maggie账户",
    "坤": "maggie账户",
    "长月": "长月账户",
    "ziqi": "ziqi账户"
  }
}
```

**字段说明：**
- `accounts`：账户名 → 目标金额（本次申购每户金额）
- `prev_funds`：上次各出资人金额
- `prev_allocation`：上次分配结果 `{"账户名": {"出资人": 金额}}`
- `new_funds`：本次各出资人金额
- `ownership`：出资人 → 所属账户名（用于差额资金的归属分配）
- `prev_title` / `new_title`：上次/本次标题
- `title`：总标题
- `note`：顶部提示语

### 2. 运行脚本

```bash
python3 scripts/transfer_diff.py input.json --png -o output/
```

参数：
- `--png`：生成 PNG 图片（需要 Playwright）
- `--html`：生成 HTML 文件
- `-o DIR`：输出目录（默认当前目录）

### 3. 输出文件

- `transfer_result.json`：分配结果 + 转账明细
- `transfer.html`：三合一 HTML 页面
- `transfer.png`：三合一 PNG 图片

### 输出图片包含三部分：
1. **上次分配表**（已完成的，作为基准）
2. **本次分配表**（不动原则，保留上次位置）
3. **转账明细**（仅差额操作：补充/新增/退款）

## 算法说明

1. **保留上次位置**：上次分配的每笔钱位置不变（不超过本次出资额）
2. **差额按归属分配**：每人多出的钱优先放归属账户
3. **完整匹配**：剩余差额尽量不拆分，整笔匹配缺口
4. **最后拆分**：实在装不下才拆分

## 图片说明

- 被拆分的出资人行底色标红，名字后带 ⚡
- 退款行橙色底
- 分节标题蓝色左边框

## 与 fund-allocation skill 的关系

- `fund-allocation`：从零开始分配，适用于首次申购
- `fund-transfer`：基于上次分配调整，适用于连续申购场景
