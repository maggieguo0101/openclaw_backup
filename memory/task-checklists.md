# 高频任务验收清单

> 目的：让 Agent 在完成任务后自检，减少"一次性"执行——老板只需确认结果，不用反复追问。
> 使用方式：Agent 在任务完成时，对照对应清单逐项确认，不满足的项目主动说明或补充。

---

## 📊 北交所打新分析

**触发条件**：用户提到北交所新股、打新、申购、920XXX

**执行前检查**：
- [ ] 是否能通过脚本自动获取发行参数？（先尝试，失败则立即告知，不超过2次重试）
- [ ] 如需人工提供数据，一次性列出所有需要的字段（发行价、发行PE、申购上限、网上发行量、申购日期）

**输出验收清单**：
- [ ] 公司基本面：主营业务、营收/净利润趋势、核心风险点
- [ ] 发行参数完整：发行价、PE、定价方式、顶格资金、正股门槛
- [ ] 获配结构：maggie / 长月 / ziqi 三账户分别能申购多少
- [ ] 三档收益率：悲观（1.5x）/ 中性（2.0x）/ 乐观（2.5x）
- [ ] 申购策略：推荐券商 + 最早时间 + 是否需要拼手速
- [ ] 首日操作策略：换手率阈值对应的处置动作
- [ ] 输出为图片（HTML → 截图 → S3Plus 链接）

**质量标准**：
- 三档年化收益率必须跑代码计算，不手算
- 顶格资金数字必须与发行公告核实（脚本估算仅供参考）
- 图片中数字不能有小数精度错误

## 📝 KM 文档分析

**触发条件**：用户提到学城、KM 链接、contentId、需要总结文档

**执行前检查**：
- [ ] 是否有 SSO 认证？（通过 CIBA 认证脚本注入 cookie）
- [ ] 文档是否超长？（超过 30000 字考虑分段读取）

**输出验收清单**：
- [ ] 核心结论：1-3 句话概括文档最重要的观点
- [ ] 与老板项目的关联：明确说明哪些内容对老板的工作有直接帮助
- [ ] 可执行建议：至少 3 条可立即行动的具体建议（不是泛泛而谈）
- [ ] 保留原文关键数据/案例（不能全部用概括代替）

**质量标准**：
- 不能只输出"文档很全面"这种无意义总结
- 建议必须有明确的行动主体和时间框架

---

## 🔍 数据分析（CSV/Excel/SQL）

**触发条件**：用户提供数据文件，要求分析、统计、洞察

**执行前检查**：
- [ ] 先输出数据基本信息（行数、字段、时间范围）再开始分析
- [ ] 确认分析目标是否清晰（不清晰先问，最多问一次）

**输出验收清单**：
- [ ] 核心结论在最前面（不超过3点）
- [ ] 数据来源和样本量明确标注
- [ ] 异常/边界情况是否已处理或说明
- [ ] 输出格式与老板需求匹配（表格/图表/长图）
- [ ] 如果是商业决策用途，给出"下一步行动建议"

**质量标准**：
- 所有数字计算必须跑代码，不手算
- 百分比加起来必须等于100%（或明确说明原因）

---

## 💬 大象/企微消息起草

**触发条件**：老板要求起草通知、公告、群消息

**输出验收清单**：
- [ ] 核心信息在第一段（谁、什么事、什么时间）
- [ ] 语气与发送场景匹配（正式/非正式）
- [ ] 长度适中（手机屏幕不超过3屏）
- [ ] 如有行动项，单独加粗或用序号标出
- [ ] 起草完主动问："需要调整语气或增减内容吗？"

---

## 🗓️ 日程/会议安排

**触发条件**：老板提到安排会议、新建日程、查忙闲

**输出验收清单**：
- [ ] 时间、地点、参与人、主题四要素完整
- [ ] 冲突检查：与现有日程是否有重叠
- [ ] 会议前提醒：是否需要设置提醒（默认15分钟前）
- [ ] 确认完成后给出日程链接或确认信息

---

> 📌 提醒：每次任务完成后，对照清单过一遍。
> 有不满足的项，主动说"X 项未完成，原因是 Y，建议 Z"——不要等老板问。
> **反馈机制（2026-03-24 老板确认）**：每次任务完成后，主动问老板"这次执行怎么样？"，把反馈记录到 `memory/agent-failures.md`，形成持续改进闭环。
