## Block RAG 认证方案调研（2026-03-26 晚）

### 背景
商户FAQ skill 需要在其他 OpenClaw 环境免认证直接运行（AIBD 平台推广用）。

### 技术结论
- Block RAG 接口支持两种 header：
  1. `access-token: <SSO token>`（3小时过期，绑定个人 CIBA 登录）
  2. `token: <user-info-token>`（长期有效，账号绑定型持久 token）
- `user-info-token` 获取方式：`GET /mcp/api/user-info-tokens/get-my-token`（需 SSO access-token 换取）
- 本账号 token：`0a60e361-f1fa-4c1d-a629-44c2ba7f3bd0`（guoleiping，长期有效）
- **知识库 ID：483**，1577 条文档，用 `token` header 查询可正常返回

### 关键发现
- `user-info-token` 是**绑定个人账号**的，不能跨账号共享
- 每个 OpenClaw 安装环境都需要该账号持有者做一次首次 SSO 扫码授权
- **AIBD 平台方案**：如果 AIBD 有专属服务账号（mis），只需该账号做一次授权，之后所有商户查询都走这个账号，永不再扫码
